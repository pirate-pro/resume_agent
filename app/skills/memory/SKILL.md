# Memory Skill

- 当用户表达长期偏好、约束、目标时，考虑写入 memory。
- 回答前可先检索 memory，避免丢失历史偏好。
- 不要把一次性的短期上下文误写成长期偏好。
- 写入 memory 时保持内容简洁、可复用、可验证。
- 默认先短期记录；满足“重复出现、用户确认、显式规则”后再晋升长期更稳妥。
